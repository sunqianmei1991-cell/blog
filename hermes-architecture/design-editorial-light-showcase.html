Agent Loop 工作流程

1

加载记忆

从 MEMORY.md 和 USER.md 加载持久化记忆快照，冻结在会话开始时。

2

组装系统提示词

组合人格定义、环境记忆、用户画像和工具使用说明，用 XML 标签结构化。

3

构建历史消息列表

系统提示词 + 对话历史 + 当前用户消息，形成完整的上下文。

4

LLM 决策

调用工具还是直接文本回复？决策中心判断下一步行动。

5

工具执行

通过注册表调用工具函数，获取执行结果，检查迭代次数防止死循环。

6

结果反馈

将工具结果追加到消息历史，继续循环或返回最终答案。

Hermes vs OpenClaw

持久化存储 无 SQLite + FTS5

核心洞察

记忆不仅仅是存储，更是上下文的延续。Hermes 的四层记忆架构让 Agent 真正具备了"记忆力"，能够跨会话保持上下文连续性。

工具注册表模式将硬编码转换为声明式配置，配合条件检查机制，实现了工具的智能加载和动态可用性判断。

代码示例：工具注册

registry.py 
class ToolRegistry:
    """中心化工具注册表"""
    
    def __init__(self):
        self._tools: dict[str, Tool] = {}
    
    def register(self, tool: Tool, check_fn: callable = None):
        # 条件检查：API Key 是否配置
        if check_fn and not check_fn():
            return  # 条件不满足，不注册
        self._tools[tool.name] = tool
    
    def get_available_tools(self) -> list[Tool]:
        return [t for t in self._tools.values() if t.is_available()]
