Anthropic ：Agent不是越复杂越高级





workflow算不算Agent？

agent能做自主规划是不是就更先进？

多agent系统是不是更高级？



很多人都陷进了一个误区——复杂度崇拜

但Anthropic 的这篇《Building Effective Agents》讲得非常清楚。

Workflow 和 Agent，本质上都属于智能体系统。

真正的区别，不在有没有大模型。

而在——决策权归谁



Workflow 

路径是人写死的。步骤是提前设计好的。模型只是执行。决策权在你手里。它可控、可复现、好调试



Agent 

路径由模型决定。过程可以动态调整。它会规划、会选择工具、会根据结果改变策略。

决策权在模型手里。但自主性一上来。不确定性也上来了。

同样的问题，今天一个答案，明天一个逻辑。企业怎么用，业务怎么验收



所以关键问题是，什么时候该用 Workflow？什么时候才需要 Agent？

三个判断标准：

路径明确、步骤固定就是流程问题

目标稳定、清洗、可枚举，那还是 流程饿问题
不需要动态策略就不要引入planning

三个都不是，才轮到Agent

不要把流程问题，做成模型问题



那最佳路径是什么？先做增强型的大模型，再做workflow，再做agent



什么是增强型的LLM

提示词写清楚，能检索，能调用基础工具，能记住上下文

很多场景，到这一步已经够了



当任务有明确步骤时用workflow

如果任务可以被拆成顺序执行的子任务——那就用提示链。

比如写营销方案。先出大纲，再写内容。

如果任务有多个处理路径。那就用路由。比如智能客服，根据用户的意图不同，是售前咨询、售后反馈，进入下游不同的路径

如果子任务彼此独立。或者你希望从多个视角分析问题，提高结果的置信度。那就并行。一起跑。最后再汇总。

如果任务变复杂，你没办法提前写好所有子步骤。那就需要一个协调器。由一个中央模型动态拆解任务，分配给不同节点，再整合结果。比如 AI Coding。改哪些文件，要根据具体需求现场决定。

如果任务需要反复优化，那就加一个评估优化器。一个模型负责生成。一个模型负责评估。

比如做 Deep Research。要不要继续检索？信息够不够？需要一个评估节点来判断



出现探索型任务，任务路径无法通过硬编码实现，或者需要多轮操作时，再引入智能体

但是要注意，在执行过程中，要有环境数据反馈，比如工具的调用结果，上一步执行的结果等

敏感问题需要引入检查点，引入人类反馈来监督执行

设置最大迭代次数，避免系统无限循环



Workflow 不是低级方案。

Agent 也不是高级标签。

不管黑猫白猫，能抓老鼠的就是好猫

复杂系统，不是设计出来的。

是从简单系统，一步一步迭代出来的。

真正拉开差距的——

不是你堆了多少 Agent。

而是你，能不能设计好决策权的边界。
