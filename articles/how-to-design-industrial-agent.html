引言：当 Agent 走出实验室

有了 vibe coding 的工具之后，做 Agent 似乎变得越来越容易：选一个大模型，写 Prompt，接上工具，再加上流程规划，代码交给 AI 来写。一个"智能体"demo 就诞生了，跑起来效果还不错。

但是当它接入真实业务场景时，当任务从单轮问答升级为多步决策，系统需要持续运行时，很多问题就会涌现出来：多步骤任务无法稳定复现，系统长期运行后状态逐渐失控。

🤔 核心问题

我们应该如何设计一个真正可上线、可扩展、可治理的 Agent 架构？

Google 的《Agentic Design Patterns》给出了答案。这本书最有价值的地方在于他是一个模式组合的思维，它把 Agent 架构拆成了 21 种模式，通过多个模式像拼图一样巧妙的组合，打造出适应各种复杂场景的智能体系统。

核心观点：非常建议每个 Agent 产品经理逐字阅读。Agent 时代，真正拉开产品经理差距的，恰恰就是 Agent 的架构设计能力。

六层架构全景图

为了更好的理解，我把这些模式按照用途分成了 6 大层级，从下往上构建一个完整的智能体系统：

可靠性治理层
Evaluation · Guardrails · Recovery · Human-in-the-Loop
让系统可控、可持续、可优化
⑤

让系统可分工、可扩展
④
外部交互层

③

让智能体记得住、跨轮持续
②

① 控制流层：让任务真正跑起来

流程控制决定了智能体如何拆解和调度任务，是将概率性的语言模型转化为具备确定性工作流的核心架构。

Prompt Chaining（提示链）

就像流水线一样，把复杂任务拆解成一系列顺序执行的步骤。每一步的输出作为下一步的输入，通过指定结构化输出（如 JSON），确保数据在步骤间精确传递，提升了多步处理的可靠性。

步骤 1：提取关键信息
↓
步骤 2：情感分析
↓
步骤 3：生成摘要

Routing（路由）

路由机制为智能体引入了条件逻辑，使智能体能根据输入上下文选择一个合适路径或工具。智能体系统突破了线性执行的限制，具备了上下文自适应能力。

用户问题 → 意图识别
↓
技术问题
→ 技术支持 Agent
账单问题
→ 财务处理 Agent

Parallelization（并行化）

并行化允许智能体同时执行多个无依赖关系的子任务，并在后续节点汇总结果，大幅缩短整体执行时间，提升效率。

用户请求：生成季度报告
↓
并行执行
统计用户行为
3s
分析市场趋势
4s
整理竞品信息
3s
↓
汇总结果 → 生成报告

Planning（规划）

上面三种是人为的规划好路径，让智能体按照既定的路径走。而 Planning 是由智能体自己来规划路径，智能体不再是一个执行者而是成为一个战略规划者，它会先分析目标，制定计划，然后再执行。

🎯 典型场景

非常典型的需要规划的场景是做 deep research：智能体先分析研究目标，自主制定调研计划，然后逐步执行并动态调整。

② 推理增强层：让智能体想得更清晰

这类模式解决的是 Agent 的"脑力"问题：如何让 LLM 想得更深入、推理更可靠。

🧠
思维链（CoT）
模型在生成最终答案之前，会先将问题拆解，逐步推导出中间步骤。通过显式推理，提升模型多步推理任务的表现。
🌳
思维树（ToT）
比思维链更进一步，探索多个分支，评估每个分支，选择最优解。
🔄
ReAct
推理与行动交替进行。边想边做，循环进行：思考-行动-观察。
🪞
Reflection（反思）
智能体具备自省能力，生成结果后自我反思答案是否合理、逻辑是否严谨。

这四种方式，思维链和思维树是前置思考，先想清楚再行动；ReAct 是过程中思考，边想边做；Reflection 是后置思考，做完了自省一下。

③ 状态与记忆层：让智能体记得住

人类有短期记忆和长期记忆，智能体也需要。智能体的记忆管理让它能够提供更个性化、更连贯的服务。这是从"一次性对话"到"持续关系"的关键。

💭
短期记忆（上下文记忆）
类似于工作记忆，保存当前正在处理或最近访问的信息。包含最近的消息、智能体回复、工具使用结果以及当前交互中的智能体反思。但这些上下文依然是临时的，会在会话结束后丢失。
🧠
长期记忆（持久记忆）
智能体需要在多次交互、任务或较长周期内保留信息的仓库，类似于长期知识库。智能体需要长期记忆中的信息时，会查询外部存储，检索相关数据，并将其整合到短期上下文中，实现知识的融合。

④ 外部交互层：接入真实世界

智能体不能只活在"文字世界"里，它需要连接真实世界。

🛠️
Tool Use（工具调用）
赋予智能体"手和脚"。它可以调用搜索引擎、计算器、数据库、API，从"只会说话"变成"能做事情"。
🔌
MCP（模型上下文协议）
标准化的连接协议，就像 USB 接口统一了各种设备。让智能体能够以标准化方式连接各种外部数据源。
📚
RAG（检索增强生成）
让 LLM 在生成响应前，先从外部知识库中提取最相关的片段以丰富上下文，减少幻觉并确保回答具有可验证的事实依据。
🤖
Agentic RAG（智能体 RAG）
进阶的 RAG 引入了推理层，智能体不仅被动接收检索结果，还会主动评估来源质量、调和信息冲突并进行多步推理。

Tool Use vs MCP：工具函数调用就像给 AI 配一套专用工具（如特定扳手和螺丝刀），适合固定任务场景。而 MCP 则像通用电源插座系统，不直接提供工具，但允许任何合规工具接入，打造动态、可扩展的智能体工作坊。

⑤ 系统协作层：可分工、可扩展

多智能体协作（Multi-Agent Collaboration）

单一的智能体在面对复杂多领域任务的时候，能力往往受限。这个时候就需要将系统结构化为多个独立的智能体，进行团队协作。

➡️
顺序交接
一个智能体完成任务后，将输出传递给下一个智能体
⚡
并行处理
多个智能体同时处理问题不同部分，结果后续合并
💬
辩论与共识
智能体基于不同视角和信息源讨论，最终达成共识或
