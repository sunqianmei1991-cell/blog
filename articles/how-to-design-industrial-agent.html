引言：当 Agent 走出实验室

2025 年，AI Agent 已经从概念验证走向了生产环境。但当我们在真实业务场景中部署 Agent 时，一个残酷的现实浮现出来：Demo 里跑得通的智能体，在持续运行、多步决策、复杂交互的场景下，往往会暴露出稳定性差、状态失控、难以扩展等问题。

Google 发布的《Agentic Design Patterns》白皮书（400+ 页）正是为了解决这些问题。这份文档的价值不在于它罗列了多少种模式，而在于它提供了一种组合式思维——将 Agent 架构拆解为 21 种可复用的设计模式，通过像拼图一样巧妙组合，打造出适应各种复杂场景的工业级智能体系统。

核心观点：对于 Agent 产品经理而言，这本书值得逐字阅读。因为在 Agent 时代，真正拉开产品经理差距的，恰恰是架构设计能力。

一、六大类别：Agent 架构的全景图

Google 将这 21 种模式按照功能划分为六大类别，每一类解决 Agent 系统中的一个核心问题：

类别 核心解决的问题 关键模式
流程控制 智能体如何把任务跑起来 Prompt Chaining、Routing、Parallelization、Planning
推理提升 智能体怎么想得更清晰、结果更可靠 CoT、ToT、ReAct、Reflection
外部交互 智能体怎么连接真实世界 Tool Use、MCP、RAG
记忆管理 保持智能体内部连贯性 短期记忆、长期记忆
协作 提升智能体解决复杂问题的能力 多智能体协作、A2A 协议
治理 确保系统安全可控 评估监控、护栏、异常处理、HITL

这六大类别构成了一个完整的 Agent 系统架构闭环：从任务启动到推理执行，从外部交互到记忆保持，从团队协作到安全治理。

二、流程控制：让任务有序运行

流程控制类模式解决的是最基础的问题：Agent 如何组织和执行任务流程。

2.1 Prompt Chaining（提示链）

核心思想：像流水线一样，将复杂任务拆解成一系列顺序执行的步骤。

步骤 1：提取文章关键信息
↓
步骤 2：进行情感分析
↓
步骤 3：生成内容摘要

MVP 阶段 低成本 固定流程

2.2 Routing（路由）

核心思想：引入条件逻辑，使智能体能根据输入上下文选择合适的路径或工具。

用户问题 → 意图识别
↓
技术问题
→ 技术支持 Agent
账单问题
→ 财务处理 Agent
投诉建议
→ 客户关系 Agent
其他问题
→ 通用客服 Agent

与 Prompt Chaining 的区别：Routing 是"分支选择"，Prompt Chaining 是"顺序执行"。

2.3 Parallelization（并行化）

核心思想：将不相干的独立子任务同时运行，提升整体效率。

用户请求：生成季度数据分析报告
↓
并行执行
子任务 A
统计用户行为
3s
子任务 B
分析市场趋势
4s
子任务 C
整理竞品信息
3s
↓
汇总结果 → 生成报告

2.4 Planning（规划）

核心思想：由智能体自主规划执行路径，从"执行者"升级为"战略规划者"。

用户输入：帮我研究一下新能源汽车市场
↓ Agent 自主规划
  1. 搜索市场规模数据
  2. 分析主要玩家（特斯拉、比亚迪等）
  3. 查找技术趋势
  4. 整理政策影响
  5. 生成综合分析报告
↓ 执行过程中动态调整计划

关键区别：前三种模式是人为规划好路径，Planning 是智能体自己规划路径。
