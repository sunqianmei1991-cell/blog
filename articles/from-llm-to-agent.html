从 LLM 到 Agent：智能客服的进化之路

给 AI 初学者的系统性指南：用一个客服助手的故事，看懂从 LLM 到 Agent 的完整进化

写在前面

如果你是刚接触 AI Agent 的产品经理，可能会被一堆术语搞得晕头转向：LLM、Prompt、RAG、Agent、上下文、记忆、编排……这些词到底是什么意思？它们之间又是什么关系？

这篇文章会用一个你熟悉的场景——智能客服助手——带你从零开始，看看 AI 是如何一步步从"只会聊天"进化到"真正能干活"的。

读完这篇文章，你会获得
  • ✅ LLM 和 Agent 的本质区别
  • ✅ Agent 的核心组成部分
  • ✅ 什么时候该用 Agent
  • ✅ Agent 在真实世界中如何运作

一、只会说话的新员工（LLM）

1.1 什么是 LLM？

LLM（Large Language Model，大语言模型）——它就像一个读过海量书籍的聪明人，能理解你的问题，用自然语言回答你。

假设你是一家电商公司的产品经理，老板让你做一个智能客服。如果你直接接入了 LLM，那它会基于训练时学到的知识乱说，一会儿像百科，一会儿像网友。

1.2 Prompt Engineering

于是你给它写了一段 Prompt（提示词），就像给新员工的工作手册。

什么是 Prompt？

Prompt 就是你给 LLM 的指令和背景信息，告诉它应该扮演什么角色、遵循什么规则。

prompt.txt 
你是 XX 电商的客服助手。

【公司政策】
- 退货政策: 7天无理由退货
- 运费规则: 满99元包邮
- 会员权益: 会员享受95折优惠

【回答规则】
1. 态度友好,称呼用户为"亲"
2. 不确定的信息不要编造
3. 复杂问题转人工客服

用户问题: {用户问题}

1.3 RAG —— 给它一个知识库

RAG（Retrieval-Augmented Generation，检索增强生成）就像给 LLM 配了一个图书管理员。当用户提问时，系统先去公司的知识库里搜索相关信息，然后把搜到的内容和用户问题一起交给 LLM。

RAG 的核心价值

✅ 知识可以无限扩展 ✅ 更新文档就自动更新知识 ✅ 回答有据可查 ✅ 成本更低

1.4 Context —— 给它一个记事本

Context 就是管理对话历史，让 LLM"记住"之前说了什么。就像给它一个记事本。

但它还不是 Agent！

你的助手已经是一个"合格的客服"了，但它只能"回答问题"，不能"执行操作"。LLM 和 Agent 真正的分水岭，是它能不能替你做事。

二、能真的帮你办事的员工（Agent）

2.1 给它"双手"（Tools）

如果用户说"帮我查一下我的订单"，这需要真正去调用订单系统的 API，而不只是查知识库。

Tools（工具）——就像给 LLM 配了一套工具箱：查订单、退款、修改地址、发优惠券……

维度 LLM Agent
能力 只能"说" 能"说"也能"做"
类比 顾问 员工
工具 无 有（API、数据库等）

2.2 给它"神经系统"（Orchestration）

Orchestration（编排）——让 Agent 能"像人一样做事"：先想想怎么做，再一步步执行。

核心模式：ReAct（Reason + Act）

ReAct Workflow 
1. Reason(推理): 分析问题,制定计划
2. Act(行动): 执行一个工具
3. Observe(观察): 看执行结果
4. 重复 1-3,直到任务完成

2.3 给它"记忆"（Memory）

类型 短期记忆 长期记忆
范围 只记住当前对话 跨对话的持久化信息
生命周期 对话结束就清空 永久保存

2.4 Agent 的完整架构

┌─────────────────────────────────────┐ │ Model(大脑) │ │ GPT-4 / Claude / Gemini │ │ 负责:理解、推理、决策 │ └─────────────────────────────────────┘ ↕ ┌─────────────────────────────────────┐ │ Orchestration(神经系统) │ │ - Planning(规划) │ │ - Memory(记忆) │ │ - Context Engineering(上下文) │ └─────────────────────────────────────┘ ↕ ┌─────────────────────────────────────┐ │ Tools(双手) │ │ - 查询订单 API │ │ - 创建退货单 API │ │ - RAG 知识库 │ └─────────────────────────────────────┘
核心洞察

从 LLM 到 Agent，本质上是从"只会说"到"能做事"的跨越。

三、从新手到专家的能力分级

Google 五级能力模型：

  • Level 0: 核心推理系统 — 纯 LLM
  • Level 1: 连接型问题解决者 — + Tools
  • Level 2: 战略型问题解决者 — + 编排
  • Level 3: 协作型多智能体系统 — 多 Agent
  • Level 4: 自我进化系统 — 研究中

四、技术演进路径

从 0 到 1 的实战建议：

  • 第一阶段：MVP — LLM + Prompt，验证需求
  • 第二阶段：知识增强 — + RAG，信息实时更新
  • 第三阶段：功能扩展 — + Tools，执行操作
  • 第四阶段：智能升级 — + Agent，处理复杂任务
  • 第五阶段：规模化 — + 编排 + 记忆，企业级应用

写在最后

从 LLM 到 Agent，技术在进化，但产品思维不变：

  • 用户需要什么？
  • 最简单的方案是什么？
  • ROI 如何？

不要被术语吓到，也不要盲目追求最新技术。理解每个概念的本质，选择适合你产品的方案，才是好的产品经理。

希望这篇文章能帮你理清 AI Agent 的技术脉络。下次再听到 LLM、RAG、Agent 这些词，你就知道它们在整个拼图中的位置了。

关键术语

  • LLM：大语言模型，AI 的"大脑"
  • Prompt：给 LLM 的指令和背景
  • RAG：检索增强生成，让 LLM 能查外部知识
  • Tool: LLM 可以调用的功能接口
  • Agent：能自主规划和执行任务的智能体
  • ReAct：思考-行动循环框架
